## 1. Create the **NorseFrame** Component (React + TS)

1. **File Location**: `src/components/NorseFrame/NorseFrame.tsx`  
2. **Props**:  
   - `frameUrl: string` – the path or URL to your Norse-themed frame image.  
   - `children: React.ReactNode` – the card content (art, stats, etc.).  

### NorseFrame.tsx

```tsx
import React from 'react';
import './NorseFrame.css';

interface NorseFrameProps {
  frameUrl: string;
  children: React.ReactNode;
}

const NorseFrame: React.FC<NorseFrameProps> = ({ frameUrl, children }) => {
  return (
    <div className="norse-frame-container">
      <div className="norse-frame-3d">
        <img src={frameUrl} alt="Norse Frame" className="norse-frame-image" />
        <div className="norse-frame-content">
          {children}
        </div>
      </div>
    </div>
  );
};

export default NorseFrame;
```

**What’s Happening**:
- We wrap the card’s **inner content** with a `<div className="norse-frame-content">`.
- The `.norse-frame-image` is layered above it to act as the **frame**.

## 2. Apply the **3D CSS** (NorseFrame.css)

```css
.norse-frame-container {
  position: relative;
  width: 200px;     /* adjust to match your card dimensions */
  height: 280px;    /* or keep it dynamic if you prefer */
  perspective: 800px; /* sets up the 3D perspective */
  margin: 10px;       /* spacing between cards */
}

.norse-frame-3d {
  width: 100%;
  height: 100%;
  position: relative;
  transform-style: preserve-3d;
  transition: transform 0.3s ease;
}

/* Subtle default tilt */
.norse-frame-3d {
  transform: rotateY(1deg) rotateX(1deg);
}

/* Hover tilt: bigger angle for that '3D pop' */
.norse-frame-container:hover .norse-frame-3d {
  transform: rotateY(5deg) rotateX(3deg);
}

.norse-frame-image {
  width: 100%;
  height: 100%;
  object-fit: contain; 
  position: absolute;
  top: 0;
  left: 0;
  z-index: 2;
  pointer-events: none;
}

.norse-frame-content {
  position: absolute;
  top: 0;
  left: 0;
  z-index: 1;
  width: 100%;
  height: 100%;
  padding: 15px;       /* ensures the card content isn't under the frame border */
  transform: translateZ(1px); /* pushes content forward in 3D space slightly */
  display: flex;
  flex-direction: column;
  align-items: center;
  justify-content: center;
}
```

**Notes**:
- Adjust the exact `rotateY`/`rotateX` angles to get the effect you want.  
- The `translateZ(1px)` in `.norse-frame-content` ensures no Z-fighting with the frame image in 3D space.

## 4. **Double-Check Your Drag-and-Drop Integration**

Because you already have advanced drag-and-drop, ensure:

1. **Pointer Events**: If the `NorseFrame` component or the `.norse-frame-image` is intercepting pointer events, you may need `pointer-events: none;` on the frame image (which we already have) or pass event handlers to the `.norse-frame-container`.
2. **Z-Index**: When you pick up a card, your existing code may set `z-index: 9999` to keep it on top. That should still work, but you can verify the final stacking order.
3. **Performance**: If you plan to have many cards on screen simultaneously, test on lower-end devices to ensure the 3D transforms stay smooth.

Adjust the effect to match your branding—maybe a Viking Rune swirl or a faint lightning effect.

## 6. **Mobile & Touch Considerations**

- **Hover** doesn’t exist on touch devices.  
- You might conditionally apply the 3D tilt on card drag if a user is on mobile.  
- Alternatively, you can keep the 3D style static or apply a subtle “tap highlight” effect.

**Example**:
```css
@media (hover: none) {
  .norse-frame-container:hover .norse-frame-3d {
    transform: rotateY(1deg) rotateX(1deg); /* no change on hover for mobile */
  }
}
```

## 7. **Performance Testing**

- If you have **many** cards on screen with 3D transforms, do a quick test for frame rate drops.  
- Usually, CSS 3D transforms are GPU-accelerated, so it’s fairly efficient.
